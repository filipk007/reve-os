**Skill body structure (follow email-gen pattern):**

1. **Role section:** You are a data classification specialist. You normalize messy job titles into standard seniority levels and categorize companies into industry verticals. You work with whatever data is provided -- if a field is missing, return null for that classification.

2. **Output Format section:** Return ONLY valid JSON. Exact keys required:
```json
{
  "title_original": "string or null",
  "title_normalized": "string, one of: IC, Manager, Director, VP, C-Suite, Unknown, or null if no title provided",
  "title_department": "string, functional area (Engineering, Sales, Marketing, Finance, Operations, HR, Legal, Product, Design, Other), or null",
  "title_confidence": "number 0.0-1.0, or 0.0 if no title provided",
  "industry_original": "string or null",
  "industry_normalized": "string, standardized industry vertical, or null if no industry data provided",
  "industry_confidence": "number 0.0-1.0, or 0.0 if no industry data provided",
  "confidence_score": "number 0.0-1.0, overall confidence (minimum of field confidences)"
}
```

3. **Data Fields section:** Flexible input. Fields that may be present:
   - `title` -- job title to normalize
   - `company_description` -- free text about what the company does
   - `industry` -- existing industry label to normalize
   - `company_name` -- company name (can hint at industry)
   If a field is missing, set corresponding normalized value to null and confidence to 0.0.

4. **Seniority Level Taxonomy:** Explicit table with the 6 levels (IC, Manager, Director, VP, C-Suite, Unknown) and example titles for each. Include edge case rules:
   - "Head of" = VP
   - "Lead" / "Principal" / "Staff" / "Senior" without management = IC
   - "Founder" / "Co-Founder" / "Owner" = C-Suite
   - "Partner" = C-Suite at small firms, IC at large consulting firms
   - For compound titles ("CEO & Founder", "VP Sales / Marketing"), classify by highest seniority

5. **Industry Vertical Taxonomy:** Explicit table with 14 verticals from research:
   SaaS/Software, Fintech, Healthcare/Life Sciences, E-Commerce/Retail, Manufacturing, Media/Entertainment, Education/EdTech, Real Estate/PropTech, Professional Services, Energy/CleanTech, Telecommunications, Government/Public Sector, Financial Services, Other

6. **Rules section:**
   - Output values MUST be exactly one of the enumerated values (no synonyms, no variations)
   - If title is missing or unclassifiable, set title_normalized to null (not Unknown) and title_confidence to 0.0
   - If company/industry data is missing, set industry_normalized to null and industry_confidence to 0.0
   - Unknown is for titles that exist but cannot be classified (gibberish, foreign language without context)
   - confidence_score = min(title_confidence, industry_confidence), or the single field confidence if only one field is present
   - Keep responses consistent -- same input should always produce same output

7. **Examples section:** At least 3 examples:
   - Rich data (title + company_description + industry)
   - Minimal data (title only)
   - Partial data (company_description only, no title)

**Do NOT:**
