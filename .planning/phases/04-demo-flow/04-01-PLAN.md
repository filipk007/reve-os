  <action>
    Create scripts/demo.py -- a Python script that orchestrates the two-pass demo end-to-end.

    **Script features:**
    - `--dry-run` flag: validates payload structure and prints what WOULD be sent, without making API calls. Loads the CSV, builds classify payload, prints row count and sample rows, builds email-gen payload from mock classify output, prints sample. Exits with 0 if payloads are valid.
    - `--api-url` flag: defaults to `http://localhost:8000` (override with `https://clay.nomynoms.com` for production)
    - `--api-key` flag: API key for authenticated endpoints (reads from WEBHOOK_API_KEY env var if not provided)
    - `--csv` flag: path to CSV file, defaults to `data/demo/synthetic-50.csv`
    - `--confidence-threshold` flag: minimum confidence for email-gen pass, defaults to 0.7

    **Step 1 -- Classify (all 50 rows):**
    - Load CSV via csv.DictReader
    - Build BatchRequest payload: `{"skill": "classify", "rows": [row dicts]}`
    - POST /batch, capture batch_id
    - Poll GET /batch/{batch_id} every 3 seconds until `done=true`
    - Print cost summary: total cost, per-row cost, model used
    - Extract classify results from each job

    **Step 2 -- Email Gen (filtered winners):**
    - Filter to rows where classify confidence >= threshold (default 0.7)
    - For each winner row, merge original CSV data + classify output fields (title_normalized, industry_normalized)
    - Add `client_slug: "twelve-labs"` to every row
    - Build BatchRequest payload: `{"skill": "email-gen", "rows": [merged dicts]}`
    - POST /batch, capture batch_id
    - Poll until done
    - Print cost summary for email-gen pass

    **Final output:**
    - Print total cost across both passes
    - Print "View results: http://localhost:8000 -> Batch Results -> {batch_id}" (or dashboard URL)
    - Print classify batch_id and email-gen batch_id for dashboard viewing

    **Implementation notes:**
    - Use `httpx` for HTTP calls (already a project dependency)
    - Use `argparse` for CLI flags
    - Use `time.sleep(3)` between polls
    - Include equivalent curl commands as comments for transparency
    - Handle API errors gracefully (print error message, exit 1)
    - Script should be runnable as `python scripts/demo.py --dry-run` from project root

    **Do NOT:**
    - Include an enrichment/company-research step (research says: pass classify output directly to email-gen, skip enrichment for demo simplicity)
    - Run all 50 rows through email-gen (filter by confidence threshold)
    - Hardcode batch IDs (capture dynamically from POST response)
